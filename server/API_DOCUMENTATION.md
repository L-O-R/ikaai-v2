# Public API Documentation

This document describes the public JSON API exposed by the IKAAI INDIA CMS server.

Current API base path:

```text
/api/
```

Example local base URL:

```text
http://127.0.0.1:8000/api/
```

The current URL configuration does not expose `/api/v1/`. If versioning is added later, update `config/urls.py`, tests, and this document together.

## General API Rules

- Public read APIs do not require authentication.
- Public inquiry submission does not require authentication.
- Admin/CMS write operations are handled through Django Admin, not public JSON APIs.
- Public project and statistic APIs return active records only.
- Request and response bodies use JSON unless the endpoint redirects to Google OAuth.
- Uploaded image fields are returned as URLs or paths generated by Django REST Framework.
- Dates and internal IDs are not exposed by current public serializers.

## Status Codes

| Status Code | Meaning |
| --- | --- |
| `200 OK` | Request succeeded. |
| `201 Created` | Resource was created successfully. |
| `400 Bad Request` | Request JSON failed validation. |
| `404 Not Found` | Requested active object does not exist. |
| `405 Method Not Allowed` | HTTP method is not supported for the endpoint. |
| `500 Internal Server Error` | Unexpected server error. |

## Error Format

Validation errors follow Django REST Framework's default error format.

Example:

```json
{
  "email": [
    "Enter a valid email address."
  ],
  "message": [
    "Message is required."
  ]
}
```

Detail errors may use this shape:

```json
{
  "detail": "Not found."
}
```

## Authentication

The public JSON endpoints listed below do not require API tokens, JWT, or sessions.

Admin authentication is separate and uses Google OAuth with Django sessions:

- `GET /admin/login/`
- `GET /accounts/google/login/`
- `GET /accounts/google/callback/`

These Google routes are browser redirect routes, not JSON APIs.

## Endpoints Summary

| Method | Endpoint | Authentication | Purpose |
| --- | --- | --- | --- |
| `GET` | `/api/projects/` | Public | List active projects. |
| `GET` | `/api/projects/{slug}/` | Public | Retrieve one active project. |
| `POST` | `/api/inquiries/` | Public | Submit an inquiry. |
| `GET` | `/api/statistics/` | Public | List active organization statistics. |

## Projects API

Projects represent public website work/case-study items.

Only projects with `is_active=True` are returned.

### List Projects

```http
GET /api/projects/
```

Returns a paginated list of active projects.

### Query Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `page` | integer | No | Page number. |
| `page_size` | integer | No | Number of items per page. Default is `12`, maximum is `48`. |
| `featured` | boolean | No | Filter by featured status. Use `true` or `false`. |
| `client` | string | No | Exact, case-insensitive client name. |
| `location` | string | No | Case-insensitive partial location match. |
| `search` | string | No | Searches project title, client name, and location. |
| `ordering` | string | No | Sort by `created_at`, `display_order`, or `title`. Prefix with `-` for descending order. |

Default ordering:

```text
display_order, title
```

### Example Requests

```http
GET /api/projects/
```

```http
GET /api/projects/?featured=true
```

```http
GET /api/projects/?client=Example%20Client
```

```http
GET /api/projects/?location=Delhi
```

```http
GET /api/projects/?search=school&page=2&page_size=6
```

```http
GET /api/projects/?ordering=-created_at
```

### Success Response

Status:

```text
200 OK
```

Response body:

```json
{
  "count": 25,
  "next": "http://127.0.0.1:8000/api/projects/?page=2",
  "previous": null,
  "results": [
    {
      "slug": "annual-impact-report",
      "title": "Annual Impact Report",
      "cover_image": "/media/projects/cover/report.png",
      "client": "Example Client",
      "client_logo": "/media/clients/logos/example.png",
      "location": "Delhi",
      "featured": true,
      "short_description": "This is a short public summary of the project."
    }
  ]
}
```

### Response Fields

| Field | Type | Nullable | Description |
| --- | --- | --- | --- |
| `count` | integer | No | Total number of matching projects. |
| `next` | string | Yes | URL for the next page. |
| `previous` | string | Yes | URL for the previous page. |
| `results` | array | No | List of project objects. |
| `results[].slug` | string | No | Unique project slug. |
| `results[].title` | string | No | Project title. |
| `results[].cover_image` | string | No | Cover image URL/path. |
| `results[].client` | string | No | Client name. |
| `results[].client_logo` | string | Yes | Client logo URL/path, or `null` if unavailable. |
| `results[].location` | string | No | Project location. May be an empty string. |
| `results[].featured` | boolean | No | Whether the project is featured. |
| `results[].short_description` | string | No | Description shortened to 160 characters. |

Status:



- Slug does not exist.
- Project exists but `is_active=False`.

## Inquiries API

The inquiry endpoint accepts public contact form submissions.

It is write-only. `GET`, `PUT`, `PATCH`, and `DELETE` are not allowed.

### Create Inquiry

```http
POST /api/inquiries/
```

### Headers

```http
Content-Type: application/json
Accept: application/json
```

### Request Body

```json
{
  "name": "Aarav Sharma",
  "email": "aarav@example.com",
  "subject": "Partnership enquiry",
  "message": "We would like to discuss a possible collaboration."
}
```

### Request Fields

| Field | Type | Required | Max Length | Validation |
| --- | --- | --- | --- | --- |
| `name` | string | Yes | `150` | Must not be blank or whitespace only. |
| `email` | string | Yes | Email length handled by DRF/Django | Must be a valid email address. |
| `subject` | string | Yes | `200` | Must not be blank or whitespace only. |
| `message` | string | Yes | No explicit max length | Must not be blank or whitespace only. |

### Success Response

Status:

```text
201 Created
```

Response body:

```json
{
  "success": true,
  "message": "Your inquiry has been submitted successfully."
}
```

### Side Effects

When an inquiry is created:

1. A new `Inquiry` record is stored.
2. `is_read` defaults to `false`.
3. `is_archived` defaults to `false`.
4. The server attempts to send a notification email if `CONTACT_NOTIFICATION_EMAIL` is configured.

Email delivery failure does not change the success response of the API request.

### Validation Error Response

Status:

```text
400 Bad Request
```

Example response:

```json
{
  "name": [
    "Name is required."
  ],
  "email": [
    "Enter a valid email address."
  ],
  "subject": [
    "Subject is required."
  ],
  "message": [
    "Message is required."
  ]
}
```

### Method Not Allowed Response

Request:

```http
GET /api/inquiries/
```

Status:

```text
405 Method Not Allowed
```

Example response:

```json
{
  "detail": "Method \"GET\" not allowed."
}
```

Only statistics with `is_active=True` are returned.

The endpoint returns a maximum of 4 statistics.

### Query Parameters

## Browser Redirect Routes

These routes are public in the sense that a browser can open them, but they are not JSON APIs.

### Admin Login Page

```http
GET /admin/login/
```

Returns the Google-only admin login page.

### Google Login Start

```http
GET /accounts/google/login/
```

Redirects the browser to Google's OAuth consent/login flow.

### Google Callback

```http
GET /accounts/google/callback/
```

Handles Google's OAuth callback, validates the user, starts a Django session, and redirects to Django Admin.

Possible outcomes:

- Success: redirects to `/admin/`.
- Expired OAuth session: redirects to `/admin/login/` with an error message.
- Unprovisioned user: redirects to `/admin/login/` with an error message.
- Inactive user: redirects to `/admin/login/` with an error message.
- Non-staff user: redirects to `/admin/login/` with an error message.
- Google subject mismatch: redirects to `/admin/login/` with an error message.

## Data Limits and Business Rules

### Featured Projects

Only 4 active featured projects are allowed.

If an admin tries to save a fifth active featured project, validation fails.

### Organization Statistics

Only 4 active organization statistics are allowed.

The public endpoint also returns at most 4 active statistics ordered by `display_order`.

### Project Slugs

Project slugs are generated from titles when the project is first saved.

If another project already uses the same slug, the server appends a numeric suffix.

Example:

```text
annual-impact-report
annual-impact-report-2
annual-impact-report-3
```

After a slug is created, changing the title does not automatically change the slug.

## Frontend Integration Notes

Recommended frontend usage:

1. Use `/api/statistics/` for homepage/statistics sections.
2. Use `/api/projects/?featured=true` for featured project sections.
3. Use `/api/projects/` for project listing pages.
4. Use `/api/projects/{slug}/` for project detail pages.
5. Submit contact forms to `/api/inquiries/`.

For image fields:

- Treat returned image strings as server-generated media URLs/paths.
- Prefix with the backend origin if the API returns a relative path.

Example:

```js
const imageUrl = new URL(project.cover_image, backendBaseUrl).toString();
```

## JavaScript Fetch Examples

### Fetch Projects

```js
async function getProjects() {
  const response = await fetch("http://127.0.0.1:8000/api/projects/");

  if (!response.ok) {
    throw new Error("Failed to fetch projects");
  }

  return response.json();
}
```

### Submit Inquiry

```js
async function submitInquiry(formData) {
  const response = await fetch("http://127.0.0.1:8000/api/inquiries/", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "Accept": "application/json"
    },
    body: JSON.stringify({
      name: formData.name,
      email: formData.email,
      subject: formData.subject,
      message: formData.message
    })
  });

  const data = await response.json();

  if (!response.ok) {
    return {
      ok: false,
      errors: data
    };
  }

  return {
    ok: true,
    data
  };
}
```
